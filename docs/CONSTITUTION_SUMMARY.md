## 🎯 確立された4つの核となる原則

### I. テスト駆動開発（TDD）- 譲歩不可
```
レッドフェーズ → グリーンフェーズ → リファクタリング
(テスト失敗) → (テスト合格) → (改善)
```
- ユニットテスト、統合テスト、E2E テスト必須
- テストは**実装の前に**記述
- Red-Green-Refactor サイクルを厳密に実施

### II. コード品質とメンテナンス性
```
✓ ESLint / Prettier による自動チェック
✓ 循環的複雑度 ≤ 10
✓ 日本語エラーメッセージ
✓ DRY 原則 + 単一責任原則
```

### III. ユーザー体験の一貫性
```
✓ デザインシステム（統一UI）
✓ WCAG 2.1 Level AA（アクセシビリティ）
✓ 日本語をメイン言語
✓ 画面遷移 300ms 以内
```

### IV. パフォーマンス要件
```
✓ FCP ≤ 1.5秒、LCP ≤ 2.5秒
✓ バンドルサイズ < 150KB (gzip)
✓ Core Web Vitals 準拠
✓ N+1 クエリ排除、WebP 画像最適化
```
