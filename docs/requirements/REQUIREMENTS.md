# 要件定義書（Requirements）

技術学習ログ蓄積・公開ブログプラットフォーム「my-main-blog」の詳細な要件定義です。

---

## 1. 背景（Background）

技術学習のログを蓄積し、自分の知識を資産化したい。  
また、蓄積した内容の一部は転職活動時のポートフォリオとして活用できる状態にしたい。

---

## 2. 目的（Why）

- 学習ログを継続的に蓄積する
- 知識を整理し、後から検索・参照できるようにする
- 公開できる記事は公開し、転職活動に活用できるようにする

---

## 3. 想定利用者（Who）

| 利用者 | 説明 |
|--------|------|
| **主な利用者** | 筆者本人 |
| **副次的な利用者** | 転職活動時に閲覧する採用担当者 / エンジニア |

---

## 4. 実現したいこと（What）

- 技術記事を投稿・蓄積できる
- 記事を公開・非公開に分けられる
- タグで整理できる
- カテゴリーで整理できる
- 記事を探しやすい（検索または分類で辿れる）
- アクセス解析を行い、PV や流入元が確認できる
- SEO 対策された状態で公開できる
- モバイルで閲覧できる

---

## 5. 必須機能（Must Have）

以下の機能は **絶対に実装が必要** です。

| ID | 機能 | 詳細 |
|----|----|-------|
| **F001** | 公開記事閲覧 | 認証なしで記事を閲覧できる |
| **F002** | 公開/非公開切り替え | 管理画面から記事の公開状態を切り替え可能 |
| **F003** | タグ機能 | 記事に複数のタグを付与、タグ一覧から検索可能 |
| **F004** | カテゴリー機能 | 記事をカテゴリー分類、カテゴリー一覧から検索可能 |
| **F005** | SEO 最適化 | メタタグ（title, description, OGP）対応 |
| **F006** | モバイル最適化 | レスポンシブデザイン対応、モバイルでの表示確認済み |
| **F007** | アクセス解析 | PV 数・流入元の記録・確認機能 |

---

## 6. 任意機能（Want to Have）

以下の機能は **工数やコストが許せば実装** する機能です。

| ID | 機能 | 詳細 | 優先度 |
|----|----|----|--------|
| **F008** | 人気記事ランキング | PV 数順にランキング表示 | 中 |
| **F009** | 全文検索 | 記事タイトル・本文から検索可能 | 低 |

---

## 7. 制約条件（Constraints）

| 制約 | 詳細 |
|-------|-------|
| **バージョン管理** | Git で管理 |
| **公開方式** | GitHub 上で公開 |
| **ランニングコスト** | 無料が理想 |
| **独自ドメイン** | 無料でなければ不要 |

---

## 8. 開発優先順位（Roadmap）

### Phase 1: MVP（Minimum Viable Product）
- ✅ ブログの基本機能（記事投稿・公開/非公開）
- ✅ タグ・カテゴリー機能
- ✅ 記事一覧・詳細ページ

### Phase 2: SEO・モバイル対応
- ⏳ メタタグ・OGP 対応
- ⏳ レスポンシブデザイン実装
- ⏳ 動作確認・テスト

### Phase 3: アクセス解析
- ⏳ アクセス解析の組み込み（Google Analytics など）
- ⏳ 管理画面での PV 確認機能

### Phase 4: 拡張機能
- ⏳ 全文検索の実装
- ⏳ 人気記事ランキング表示
- ⏳ コメント機能（検討中）

---

## 9. 非機能要件（Non-Functional Requirements）

| 要件 | 詳細 |
|-------|-------|
| **性能** | 記事一覧ページ読込時間 2 秒以下 |
| **可用性** | 稼働率 99% 以上 |
| **セキュリティ** | HTTPS 対応、認証機能保護 |
| **保守性** | コード可読性確保、ドキュメント整備 |
| **拡張性** | 新機能追加が容易な設計 |
