# speckit 日本語出力ポリシー

**最終更新**: 2026年2月14日  
**プロジェクト**: my-main-blog  
**対象**: すべての speckit コマンド出力

## ポリシー概要

このプロジェクトでは、`/speckit.plan`、`/speckit.spec`、`/speckit.tasks` のすべてのコマンドが**日本語 (日本語)** で出力されるべきです。

これはプロジェクト憲法（`Constitution v1.0.0`）の言語要件に基づいており、すべての開発ドキュメント、ユーザーメッセージ、コメントは日本語で作成されます。

## 実装方法

### 1. テンプレートファイルの日本語化

すべての speckit テンプレートの先頭に以下の言語指示を記載してあります：

```markdown
<!-- ⚠️ OUTPUT LANGUAGE: All content MUST be in 日本語 (Japanese) - required by project constitution -->
```

テンプレート位置：
- `.specify/templates/plan-template.md`
- `.specify/templates/spec-template.md`
- `.specify/templates/tasks-template.md`

### 2. 各テンプレートの日本語セクション見出し

| テンプレート | 日本語化済みセクション |
|------------|-------------------|
| **plan-template.md** | 実装計画、概要、技術的コンテキスト、プロジェクト構造、複雑性トラッキング |
| **spec-template.md** | 機能仕様書、ユーザーシナリオ、ユーザーストーリー、要件、成功基準 |
| **tasks-template.md** | タスク、タスク形式、タスクカテゴリ、パス規約 |

## 使用例

### `/speckit.plan` の出力例

```
# 実装計画: [FEATURE]

**ブランチ**: `[###-feature-name]` | **日時**: [DATE] | **仕様**: [link]
**入力**: 機能仕様書: `/specs/[###-feature-name]/spec.md`

## 概要
[機能仕様書から抽出: 主要な要件 + リサーチからの技術的アプローチ]

## 技術的コンテキスト
...
```

### `/speckit.spec` の出力例

```
# 機能仕様書: [FEATURE NAME]

**機能ブランチ**: `[###-feature-name]`
**作成日**: [DATE]
**ステータス**: ドラフト
**入力**: ユーザー説明: "$ARGUMENTS"

## ユーザーシナリオとテスト (必須)
...
```

### `/speckit.tasks` の出力例

```
# タスク: [FEATURE NAME]

**入力**: `/specs/[###-feature-name]/` からの設計ドキュメント

⚠️ **重要: テストは憲法 I (TDD) に従って最初に作成する必須** — テストタスクは常に実装タスクより前に来ます。Red-Green-Refactor サイクルは譲歩不可です。

## タスクカテゴリ (憲法に基づく)
...
```

## 遵守ガイドライン

1. **すべてのセクション見出しを日本語化**
   - 自動生成されるセクション見出しは日本語である必要があります

2. **プレースホルダーテキストも日本語で提供**
   - `[Describe this user journey in plain language]` → `[このユーザーのジャーニーを平易な言葉で説明してください]`
   - `[Measurable metric, e.g.]` → `[測定可能な指標、例:]`

3. **技術用語は英語のままでも良い**
   - 例: API、HTTP、JSON、CSS、Docker など、一般的な技術用語は英語で構わません
   - しかし説明は日本語で提供してください

4. **プロジェクト固有の用語は日本語化**
   - "User Story" → "ユーザーストーリー"
   - "Test-Driven Development" → "テスト駆動開発"
   - "Constitution Check" → "憲法チェック"

## 憲法との関連性

このポリシーはプロジェクト憲法 v1.0.0 の以下の要件に基づいています：

**Principle III - User Experience Consistency**
> - **多言語対応** — 日本語をメイン言語とし、将来の多言語化に備えた構造

**Language Policy** (憲法から抜粋)
> - プロジェクト内のすべてのドキュメント、コメント、ユーザーメッセージは日本語（日本語）である
> - TypeScript/JavaScript でのコード内コメントも日本語で記述する

## トラブルシューティング

**Q: speckit コマンドが英語で出力された場合どうすればよい？**

A: テンプレートファイルが正しく日本語化されていることを確認してください：
1. テンプレートファイルの先頭に言語指示コメントがあるか確認
2. セクション見出しが日本語になっているか確認
3. キャッシュをクリアして VS Code を再起動してください

**Q: 新しいテンプレートセクションを追加する場合は？**

A: 新しいセクションを追加する際は、以下のルールに従ってください：
1. セクション見出しを日本語にする
2. プレースホルダーテキストを日本語で提供する
3. コメントと説明もすべて日本語にする

## 次のステップ

初回の `speckit` コマンド実行時に、以下を確認してください：

1. `/speckit.plan` を実行して、出力が日本語か確認
2. `/speckit.spec` を実行して、出力が日本語か確認
3. `/speckit.tasks` を実行して、出力が日本語か確認

すべてのセクション見出しと指示が日本語で表示されることが期待されます。
